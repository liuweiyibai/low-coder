# 页面发布功能使用指南

## 功能概述

Low-Coder 平台支持将编辑完成的页面发布为公开访问的链接，无需登录即可查看。

## 发布流程

### 1. 完成页面设计

在编辑器中完成页面的设计和配置：
- 拖拽组件到画布
- 配置组件属性
- 设置样式和交互
- 使用预览功能查看效果

### 2. 点击发布按钮

在顶部工具栏右侧找到绿色的 **"发布"** 按钮并点击。

### 3. 配置发布信息

在弹出的发布对话框中：
- **页面名称**: 输入易于识别的页面名称
- 点击 **"发布"** 按钮确认

### 4. 获取发布链接

发布成功后：
- 系统会生成唯一的访问链接（格式：`https://your-domain.com/p/{id}`）
- 可以点击 **"复制"** 按钮复制链接
- 可以点击 **"访问页面"** 按钮在新标签页查看

### 5. 分享页面

将获取的链接分享给其他人，他们可以直接访问页面，无需登录。

## 功能特点

### ✅ 即时发布
- 点击发布后立即生效
- 无需等待构建或部署
- 实时更新内容

### ✅ 独立访问
- 每个页面有唯一的访问链接
- 不需要登录即可查看
- 支持移动端和桌面端访问

### ✅ 持久化存储
- 发布的页面永久保存
- 链接长期有效
- 可以随时更新内容

## 页面 URL 格式

```
https://your-domain.com/p/{pageId}
```

示例：
```
https://localhost:3000/p/abc123XYZ
```

## 技术实现

### API 端点

#### 发布页面
```
POST /api/publish
Content-Type: application/json

{
  "name": "活动报名页面",
  "schema": {...}
}

响应：
{
  "success": true,
  "data": {
    "id": "abc123XYZ",
    "name": "活动报名页面",
    "url": "/p/abc123XYZ",
    "publishedAt": "2026-02-12T10:00:00.000Z"
  }
}
```

#### 访问页面
```
GET /api/publish?id={pageId}

响应：
{
  "success": true,
  "data": {
    "id": "abc123XYZ",
    "name": "活动报名页面",
    "schema": {...},
    "publishedAt": "2026-02-12T10:00:00.000Z"
  }
}
```

### 前端路由

#### 编辑器页面
```
/ - 主编辑器界面
/preview - 预览页面（临时）
```

#### 公开访问页面
```
/p/[id] - 已发布的页面
```

## 注意事项

### 🔒 安全性
- 当前版本使用内存存储（重启后丢失）
- 生产环境需要连接数据库
- 考虑添加访问控制和权限管理

### 📊 性能优化
- 页面使用 SSR 渲染，首屏加载快
- 组件按需加载
- 建议优化大型图片资源

### 🔄 内容更新
- 当前版本不支持更新已发布页面
- 需要重新发布生成新链接
- 计划添加版本管理功能

## 常见问题

### Q: 发布的页面可以修改吗？
A: 当前版本不支持修改，需要重新发布。未来版本会添加更新功能。

### Q: 页面链接会过期吗？
A: 链接永久有效，除非手动删除页面。

### Q: 可以统计页面访问量吗？
A: 当前版本暂不支持，计划在未来版本中添加。

### Q: 如何删除已发布的页面？
A: 当前版本需要在服务器端删除。计划添加页面管理界面。

### Q: 发布失败怎么办？
A: 检查浏览器控制台的错误信息，确保网络连接正常。如果问题持续，请联系技术支持。
